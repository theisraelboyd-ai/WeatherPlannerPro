📖 OPERATION PLANNER - QUICK START

1. Load Your Data

Click 📁 LOAD FILES in the top-left to upload your .js file containing your operability limits matrix.

2. Build the Sequence

In the left sidebar, enter a Task Name, select your primary/secondary analysis, heading tolerance, and duration. Add it to the timeline. Double-click any task block on the timeline to edit it.

3. Set the Weather

In the right sidebar, input Wind, Sea, Swell, and Current parameters. Save the block to overlay the forecast against your operational timeline.

4. Run the Simulation

Use the timeline at the bottom to scrub through time, or hit ▶ RUN SIMULATION to watch the vessel automatically seek the optimal safe heading within your set tolerances.

  • Green Heatmap: Safe heading.
  • Orange Heatmap: Warning (approaching limits).
  • Red Heatmap: Limit exceeded (Unsafe).

5. Print the Report

Click 🖨️ PRINT PLAN to generate a clean, PDF-ready summary report. It includes sequence tables, limit validations, and visual radar summaries for each stage.

📊 REQUIRED DATA FORMAT

The Operation Planner can read any `.js` or `.json` file, regardless of its filename, as long as it contains the window.masterData object with the correct matrix structure.

Rules:

  • The array must contain exactly 12 values representing Relative Directions: 0°, 30°, 60°, 90°, 120°, 150°, 180°, 210°, 240°, 270°, 300°, 330°.
  • Use null for headings that have no data or are explicitly restricted.
  • The keys (e.g., "4", "5", "6") represent the Wave Period (Tp/Ts) in seconds.
window.masterData = { "MY_CUSTOM_ANALYSIS": { "WAVE_HS_BY_TP": { "4": [ 1.2, 1.2, 1.1, 0.8, 0.6, 0.5, 0.5, 0.5, 0.6, 0.8, 1.1, 1.2], "5": [ 1.5, 1.4, 1.2, 0.9, 0.7, 0.6, 0.6, 0.6, 0.7, 0.9, 1.2, 1.4], "6": [null, null, 1.0, 0.8, 0.5, null, null, null, 0.5, 0.8, 1.0, null] } }, "ANOTHER_ANALYSIS": { "WAVE_HS_BY_TP": { // ... same structure ... } } };
